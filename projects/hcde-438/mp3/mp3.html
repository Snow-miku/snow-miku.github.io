Overview and Usage

Link to my game

Tap Hero (Simple Ver.):

This game is a battle game similar to "click hero" where the user just need to simple progress the attack button in order to damage the enemy. Click the start button to start a game. A battle theme song will be played. The goal is to attack the enemy and try to empty its health bar. Once the enemy's hp reaches zero, a winning music will be played, and a prompt will pop up to indicate your winning. After the music, the game will be initialized to its start menu.

Development Process

  1. STPE 1: Download node.js on the device. Create a new git hub resporitory and set to public.

  2. STPE 2: Initialize the `package.jspn` with `npm init`. Install the p5.play library.

  3. STPE 3: Create an index.html file with the copied template. Also, create the index.js file with the basic setup() and empty draw() funcitons.

  4. STPE 4: Install a local dev server, Rollup, the node-resolve plugin, the copy plugin, and the gh-pages package using npm.

  5. STPE 5: In package.json, copied the three scripts corresponded to the npm package installed:
    • "start": "web-dev-server --node-resolve --open --watch"
    • "build": "rollup --config"
    • "deploy": "npm run build && gh-pages -d dist"

  6. STPE 6: Create folders for media. Prepare, download, rename, and organize the files.

  7. STPE 7: In index.js, write helper functions for creating the necessary sprites with p5.play and draw them at the proper coordinates.

  10. STPE 10: Create a configuration file for Rollup called rollup.config.js. Add necessary elements under copy plugin and `treeshake: false`.

Issue Deep-Dive

Issue: As I tried to implement the health bar and make it have the effect of constantly reflecting the hp lost, I struggled on how to make the lost health only got deducted from the right side.

Solviong Process: Initially I only have one sprite representing the health bar, and I could not find any method that can only changes the color fill within the shape. Eventually, I decided to do two sprites the stacks on each other so one could be the outline and the other be the actuall health. However, it was hard to display the health correctly since the sprite are coordinated at the center in stead of the left corner. I managed to do the calculation of the current health percentage, and multiplied that with a saved original healthbar length. I also need to devided the position shift by two since it is referred to the center.
