Eternal Dungeon

Link to my game

This game is the completed version from the prototype (see fp1). It is a battle game that allow user to pick a character and beat enemy in a turn-based strategic form. The goal is to attack the enemy and empty its health bar. Once the enemy's hp reaches zero, a winning music will be played, and the player can choose to resume to the next level. The character can level up and the enemy will increase in difficulty as well.

Following is are screenshots of the game menu and in-game view.

Development process

  1. STPE 1: Create new repository for the prototype, and migrate fp1 into the new github repository.

  2. STPE 2: Initialize the `package.json` with `npm install`. It will install all the packages required for the project.

    3. Following is the middle process before the enemy attack damage is implemented.

  3. STPE 3: Added all the sprite necessary for showing damage towards the player, including prompts and hit effect.

  4. STPE 4: Implement the mechanism so that right now player can be defeated and instruct player to retry the game accordingly.

  5. STPE 5: Implement the critical attack indication, which is main factor to determine whether player will win in the battle.

  6. STPE 6: Add all the new stylings and media, including new fonts, wallpaper, sound effects, and background musics. Apply them into the game at the proper phase.

  7. STPE 7: Add new characters into the selection menu.

  8. STPE 8: Test out all the behavior so that they are properly functioning. Finish the writeup and deploy the game to github.

Issue Deep-Dive

Issue: Since I have many sprites functioned as buttons, and most them are centered at the page, many sprites overlayed and collided with each other and cause unwanted behavior such as altering the sprite direction.

Solviong Process: I tried many ways to solve it. First is slightly change the y coordinate position of each button so they are adjusted to prevent overlay. However, the total numbers of buttons were too many for this to work. I also tried to change the depth (does not work) and set collider to `none` (it will make these sprites unclickable), neither of which worked. Eventually, I combined some of the button sprites together by changing their text and function constantly.

Ideas and Future

  • Add selection and fetching for different characters options.
